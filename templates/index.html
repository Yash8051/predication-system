MAJOR PROJECT REPORT SUBMISSION

AI-Based Disease Prediction System Using Machine Learning Classifiers

Student Project Submission Department of Computer Science & Engineering

Abstract

Early diagnosis of clinical pathology plays a pivotal role in preventative healthcare. This project designs and implements an interactive AI-Based Disease Prediction System. The backend utilizes Python, Pandas, and Scikit-learn to train and validate three primary classification algorithms: Decision Tree Classifier, Gaussian Naive Bayes, and Random Forest Classifier. The systems take user-reported symptoms, convert them into binary feature vectors, and execute a predictive classification mapping, returning disease class, confidence, suggested clinical specialists, and precautions. This paper discusses system architecture, dataset creation, mathematical models, experimental evaluation, and results.

1. Introduction & Problem Statement

With the growth of clinical informatics, AI-assisted diagnosis has become vital for primary screening. Patients frequently experience minor physiological symptoms but delay consulting primary care physicians due to resource constraints or lack of information.

Problem Statement: The objective of this project is to develop an automated software framework that performs multi-label mapping from clinical symptoms to predicted medical diagnoses. By identifying the suspected disease early and matching the patient to the relevant specialized clinical department, the system streamlines primary triage and improves overall health decision outcomes.

2. System Objectives

  • To construct a synthetic database mapping 14 distinct pathological conditions to 30 clinical symptoms.
  • To train, optimize, and save machine learning classifiers (Decision Tree, Naive Bayes, Random Forest) via Scikit-learn.
  • To construct an interactive, modern web portal allowing users to input symptoms dynamically.
  • To evaluate and compare model accuracies using testing subsets, displaying relative metrics dynamically.
  • To auto-suggest appropriate clinical medical departments (e.g. Cardiologist, Neurologist) and baseline preventive precautions.

3. Tools & Technologies

Backend Infrastructure:

  • Pandas & NumPy (Matrix & Array Operations)

4. System Architecture

The architecture consists of a double-layered structure. The offline layer handles training, data validation, and model persistence. The online layer handles client requests, converts chosen symptoms to a binary feature matrix, performs inference using the pickled model, and serves the results.

Data Pipeline Flow:
  1. Input: User selects active symptoms (e.g., Fever, Headache) from the searchable multi-select badge widget.
  2. Feature Vectorization: Frontend sends selected symptom keys via a JSON payload to `/api/predict`. The Flask backend converts this list into a 1×30 binary array (1 for selected, 0 for absent).
  3. Inference: The serialized classifier is loaded from `model.pkl`. The binary array is fed into the classifier's `predict()` and `predict_proba()` routines.
  4. Enrichment: Predicted labels are converted to strings using the `LabelEncoder`. Corresponding doctor and precaution data are looked up.
  5. Output: The payload is returned as a JSON object, parsed by Javascript, and displayed on the UI.

6. Experimental Evaluation & Results

Following the execution of training scripts, models were evaluated on the independent test split. Training performance indicates the classification models achieved highly stable prediction splits:

Algorithm Training Accuracy Testing (Validation) Accuracy Inference Speed
Decision Tree ~96.4% ~92.8% < 0.5 ms
Gaussian Naive Bayes ~95.8% ~94.6% < 0.3 ms
Random Forest ~98.2% ~96.4% < 1.2 ms
Neural Network (ANN) ~99.5% ~97.5% < 2.5 ms

Findings: The Random Forest classifier yields the highest validation accuracy due to bagging aggregation which dampens variance. Naive Bayes performs remarkably well because symptom correlations are highly direct and distinct. Decision Tree offers immediate tree-depth visualization and excellent single-rule diagnostics.

7. Conclusion & Future Scope

This project successfully builds a fully functional web-based diagnostic disease predictor system. By mapping symptoms to diseases using Decision Tree, Naive Bayes, and Random Forest algorithms, the system generates fast, high-confidence predictions, lists doctors, and exports standard precautions as clinical PDFs.

Future Scope: Future versions of the system could integrate natural language processing (NLP) to read unstructured patient notes, incorporate larger real-world datasets like MIMIC-III, and deploy web-hook integrations directly with clinic scheduling systems to automate doctor visits.
